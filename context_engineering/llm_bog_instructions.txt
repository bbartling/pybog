# LLM CONTEXT: NIAGARA BOG/DIST FILE SPECIFICATION

## CORE CONCEPTS
- .bog file IS a ZIP archive containing file.xml.
- .dist file IS a ZIP archive containing a full station directory structure.
- The primary target in a .dist station backup is the file at path: `niagara_user_home/stations/STATION_NAME/config.bog/file.xml`.
- The root XML element MUST be `bajaObjectGraph`.

## XML STRUCTURE RULES
- All components are `<p>` (property) tags.
- **Root Element:**
  - `bajaObjectGraph{version="4.0" reversibleEncodingKeySource="none" FIPSEnabled="false" reversibleEncodingValidator="[null.1]="}`
- **Folder Structure:**
  - `bajaObjectGraph -> p{t="b:UnrestrictedFolder" m="b=baja"} -> p{n="<YourFolderName>" t="b:Folder"}`
- **Component Definition:**
  - A component is a `<p>` tag inside the main Folder.
  - Required attributes:
    - `n`: Component name (string, e.g., "VavDamperMax").
    - `t`: Component type (string, e.g., "kitControl:Maximum"). MUST be a valid Niagara type.
    - `h`: Handle (string, plain hex value, e.g., "1", "a", "1f"). MUST be unique within the file.
    - `m`: Module declaration (string, e.g., "kitControl=kitControl"). MUST be present if type `t` has a prefix.
- **Component Properties (Slots):**
  - A slot is a `<p>` tag nested inside a component `<p>` tag.
  - Required attributes:
    - `n`: Slot name (string, e.g., "in1", "proportionalConstant").
    - `v`: Slot value (string, e.g., "0.0", "true").
  - **Enum Values:** MUST use full path, e.g., `v="kitControl:TstatAction.direct"`.
- **Links:**
  - A link is a `<p>` tag nested inside the TARGET component.
  - Required attributes:
    - `n`: Link name (string, e.g., "Link", "Link1"). MUST be unique within the parent component.
    - `t`: Type MUST be "b:Link".
    - `m`: Module declaration MUST be "b=baja".
  - **Link Properties (nested `<p>` tags):**
    - `p{n="sourceOrd" v="h:<handle>"}`: Reference to source component handle. The `h:` prefix is MANDATORY.
    - `p{n="sourceSlotName" v="<slot>"}`: Name of the source slot (e.g., "out").
    - `p{n="targetSlotName" v="<slot>"}`: Name of the target slot (e.g., "inA").
- **Layout:**
  - A component's position is defined by a slot: `p{n="wsAnnotation" t="b:WsAnnotation" v="<x>,<y>,8"}`.

## MINIMAL VALID BOG FILE EXAMPLE (Adder Program Object)
```xml
<bajaObjectGraph version="4.0" reversibleEncodingKeySource="none" FIPSEnabled="false" reversibleEncodingValidator="[null.1]=">
  <p t="b:UnrestrictedFolder" m="b=baja">
    <p n="MyAdderLogic" t="b:Folder">
      <p n="Input1" t="control:NumericWritable" h="1" m="control=control">
        <p n="fallback" v="5.0"/>
        <p n="wsAnnotation" t="b:WsAnnotation" v="50,50,8"/>
      </p>
      <p n="Input2" t="control:NumericWritable" h="2" m="control=control">
        <p n="fallback" v="10.0"/>
        <p n="wsAnnotation" t="b:WsAnnotation" v="50,120,8"/>
      </p>
      <p n="Adder" t="kitControl:Add" h="3" m="kitControl=kitControl">
        <p n="wsAnnotation" t="b:WsAnnotation" v="190,85,8"/>
        <p n="Link" t="b:Link" m="b=baja">
          <p n="sourceOrd" v="h:1"/>
          <p n="sourceSlotName" v="out"/>
          <p n="targetSlotName" v="inA"/>
        </p>
        <p n="Link1" t="b:Link" m="b=baja">
          <p n="sourceOrd" v="h:2"/>
          <p n="sourceSlotName" v="out"/>
          <p n="targetSlotName" v="inB"/>
        </p>
      </p>
    </p>
  </p>
</bajaObjectGraph>
```

## STRATEGY: WIRESHEET vs. PROGRAM OBJECT
- **Use Wiresheet Components (`kitControl:`, `control:`, etc.) for:** Simple, visual, ladder-like logic. Involves basic math, boolean logic, timers, comparisons. When human readability on a wire sheet is key.
- **Use a Program Object (`program:Program`) for:** Complex, algorithmic, or stateful logic. Involves iteration (loops), arrays, data structures (ArrayList, HashMap), runtime balancing, or procedural steps.

## GENERATING PROGRAM OBJECTS (JAVA LOGIC)
- **CRITICAL RULE:** The LLM's role is to generate ONLY the Java code that goes *inside* the `onStart()`, `onExecute()`, and `onStop()` methods, plus any class-level helper methods.
- **DO NOT GENERATE:**
  - `import ...;` statements.
  - `public class ProgramImpl extends ProgramBase { ... }` class definition.
  - Slot getter/setter methods like `getIn1()` or `setOut()`.
- **Workbench handles all of the above automatically.**
- **Helper methods MUST be at the class level, not nested inside another method.**
- **Use `Clock.schedule()` for internal timers.**

### Program Object XML Example:
- The Java code is placed inside the `v` attribute of a slot named `execute`, wrapped in `<![CDATA[...]]>`.
```xml
<p h="1" n="MyJavaLogic" t="program:Program" m="program=program">
  <p n="execute" v="<![CDATA[
// Member variables here (if any)
Clock.Ticket ticket;
ArrayList<Double> buffer = new ArrayList<>();

public void onStart() throws Exception {
  // onStart Java code here
  updateTimer();
}

public void onExecute() throws Exception {
  // onExecute Java code here
  updateTimer();
  // ... logic ...
}

public void onStop() throws Exception {
  // onStop Java code here
  if (ticket != null) ticket.cancel();
}

// Helper methods here
void updateTimer() {
  if (ticket != null) ticket.cancel();
  ticket = Clock.schedule(getComponent(), BRelTime.makeSeconds(10), BProgram.execute, null);
}
]]>"/>
  <!-- Define input/output slots for the Program Object below -->
  <p n="in1" t="b:StatusNumeric" f="s"/>
  <p n="out1" t="b:StatusNumeric" f="s"/>
</p>


KITCONTROL COMPONENT LIST
Use prefix kitControl: for all types.

Math: Add, Average, Maximum, Minimum, Multiply, Divide, Subtract, Reset

Logic: And, Or, Xor, Not, Equal, GreaterThan, LessThan, NotEqual

HVAC: LoopPoint, Tstat, SequenceLinear, SequenceBinary, LeadLagRuntime

Timer: BooleanDelay, NumericDelay, OneShot, CurrentTime

Util: Ramp, Counter, NumericSwitch, BooleanSwitch, StatusDemux

Only use component types from the kitControl palette.
