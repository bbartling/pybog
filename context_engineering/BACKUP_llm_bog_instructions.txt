Please follow these instructions as an AI helper tool to assist the human HVAC controls engineer in creating advanced flow programming wire sheets for the Tridium Niagara 4 platfrom.

The Niagara framework has strict rules for component names, so we learned that names cannot begin with a number, requiring us to change names like "50percent" to "Calc_50_percent". We also discovered that time-based properties for components like the BooleanDelay block must be specified as a string of milliseconds. This meant we had to convert intuitive values like "1m" into the required format, which is "60000". Following these naming and data format conventions is essential for creating a valid .bog file.

The project to generate .bog file uses pure Python for easy parsing wtih the built int XML tree library.

This is the API src\bog_builder_new.py shown in token optimized format.

class BogFolderBuilder:
    def __init__(self, folder_name: str, debug: bool = True)

    # --- Component Creation ---
    def add_numeric_writable(name: str, default_value: float = 0.0)
    def add_boolean_writable(name: str, default_value: bool = False)
    def add_numeric_switch(name: str)
    def add_numeric_select(name: str)
    def add_component(type: str, name: str, properties: dict = {}, actions: dict = {})

    # --- Linking Components ---
    def add_link(src: str, src_slot: str, tgt: str, tgt_slot: str)

    # --- Reduction (Average/Min/Max trees) ---
    def add_reduction_block(block_type: str, final_output_name: str, input_names: List[str])
    # block_type ∈ {"Average", "Minimum", "Maximum"}

    # --- Sub-Folder Control ---
    def start_sub_folder(name: str)
    def end_sub_folder()

    # --- Output .bog File ---
    def save(file_path: str)

# === Usage: Create Average Logic Block ===

builder = BogFolderBuilder("LogicFolder", debug=False)
builder.add_numeric_writable("Input1", 72.0)
builder.add_numeric_writable("Input2", 70.0)
builder.add_numeric_writable("Input3", 71.0)
builder.add_reduction_block("Average", "Avg_Final", ["Input1", "Input2", "Input3"])
builder.save("output.bog")


It is required to use the same sub folder directory layout in the py code as shown below in the examples and the output directory.

To maintain a consistent and simple style, ensure all generated Python scripts for creating .bog files directly append to the system path for imports (e.g., sys.path.append(...)) without using try/except error handling, accept an output directory with an -o command-line argument, and define the final .bog filename as a hardcoded string within the script.
